结构化文档上使用图卷积神经网络进行信息提取

概述（图）

将结构化文档转换为图形的代码，并对GCN实施进行全面调查以对其中的文本实体进行分类

诸如CNN之类的深度学习解决方案有效地捕获了欧几里得空间中数据的模式，越来越多的应用程序以图的形式表示数据并且缺乏网格状规则性。 由于图可能是不规则的，它们可能具有可变大小的无序节点，并且每个节点可能具有不同数量的邻居，从而导致数学运算（例如卷积）难以应用于图域。

此类非欧几里得数据的一些示例包括：

  • 蛋白质互作用数据：分子之间的相互作用被建模成图
  • 引用网络：论文是节点，引文是单向或双向边沿
  • 社交网络：人是节点，他们的关系是边

本文特别讨论了在诸如发票和票据之类的结构化文档上使用图卷积神经网络（GCN），以通过学习文本实体之间的位置关系来自动提取有意义的信息。 讨论了多种方法以及将文档转换为图形的方法，因为文档本身并不具有类似图形的布局。

图表示为G = （V, E），其中V：节点的集合；E：边的集合

如何将结构化文档转为图

当涉及诸如标题和表格之类的实体时，诸如发票和票据之类的结构化文档具有一定的出现顺序。 例如，在大多数发票中，表格位于页眉和页脚之间。 另外，总金额（是我们希望提取的发票的重要实体）通常位于表格的右下角。 此类重复出现的结构信息以及文本属性可以帮助图神经网络学习邻域表示并进行结点分类。

那么，我们如何以图的形式来获取此类文档呢？答案是结合几何算法与OCR

OCR

在此步骤中，我们使用市售的OCR引擎（例如Amazon Textract或Google Vision API）来生成所谓的文档对象映射。

每一个文档对象映射都是一个文本实体，一个单词或一个句子。 在本文的上下文中，我们将对象映射称为Pandas DataFrame，它具有五列：xmin，ymin，xmax，ymax，Object。

（xmin，ymin）和（xmax，ymax）给我们矩形边界框的坐标，而Object给出该框内的实际文本。

下面是使用Google Vision API实例，用来生成此类Object Map的github上的一份代码：

import os
import cv2
from itertools import chain
import base64
import pandas as pd
import requests
import json

def ocr_using_google_api(image_path, request_url):
    '''
    This function uses Google Vision API for Text Detection

    Args :
        image_path : Input image path

    Returns:
        pd.DataFrame having coordinates of each text box along with detected
        text inside it.
    '''
    lstr_filename, str_extension = os.path.splitext(str(image_path))

    image_arr = cv2.imread(image_path)
    _, image_buffer = cv2.imencode("."+str_extension,
                                            image_arr)

    int_respose_code = 0
    json_request_header = {
        'content-type': 'application/json',
        'Accept-Charset': 'UTF-8'
    }

    str_encode_image = base64.b64encode(image_buffer).decode()
    json_request_payload = {'requests':
            [
                {
                    "image":
                    {
                        'content':str_encode_image
                    },
                    'features':
                    [
                        {
                            'type': 'DOCUMENT_TEXT_DETECTION'
                        }
                    ],
                }
            ]
        }

    list_block_coordinates = []
    list_block_word_coordinates = []
    list_each_word_coordinate = []

    str_http_response = \
        requests.post(
            request_url,
            data=json.dumps(json_request_payload),
            headers=json_request_header,
            verify=False
        )

    int_respose_code = str_http_response.status_code
    if int_respose_code != 200:
        return list_block_coordinates

    else:
        json_response_data = json.loads(str_http_response.text)
        if json_response_data['responses'][0]:

            list_bounding_boxes = \
                json_response_data['responses'][0]['fullTextAnnotation']\
                    ['pages'][0]['blocks']

            list_vertices = \
                [boundingBox['boundingBox'] for boundingBox in \
                    list_bounding_boxes if 'boundingBox' in boundingBox]

            list_block_coordinates = \
                [list(chain(*[[x['x'], x['y']] for x in i['vertices']])) \
                    for i in list_vertices]

            list_block_words = []
            for bounding_box in list_bounding_boxes:
                list_paragraphs = bounding_box["paragraphs"]
                str_word = ""
                list_bounding_box = []

                for paragraphs in list_paragraphs:
                    list_words = paragraphs['words']
                    for words in list_words:
                        list_vertices = []
                        str_text = ""
                        llst_symbols = words['symbols']
                        list_bounding_box.append(words['boundingBox'])

                        for symbols in llst_symbols:
                            str_text = (str_text + symbols['text']).strip()
                            list_vertices.append(symbols['boundingBox'])

                        str_word = (str_word + " " + str_text).strip()
                        list_word_coords = \
                            list(chain(*[[x['x'], x['y']] for x in \
                                words['boundingBox']['vertices']]))

                        list_word_coords.insert(0, str_text)
                        list_each_word_coordinate.append(list_word_coords)

                    list_word_coordinates = \
                        [list(chain(*[[x['x'], x['y']] for x in \
                            i['vertices']])) for i in list_bounding_box]

                list_block_words.append(str_word)
                list_block_word_coordinates.append(list_word_coordinates)

        for int_index, llst_block_coordinate in enumerate(list_block_coordinates):
            llst_block_coordinate.insert(0, list_block_words[int_index])

        list_word_objects = \
            [[min(item[1],item[5]), min(item[2],item[6]),
            max(item[1],item[5]), max(item[2],item[6]),
            item[0]] for i, item in enumerate(list_each_word_coordinate)]

        return df = pd.DataFrame(list_word_objects,
                            columns=['xmin', 'ymin', 'xmax', 'ymax', 'Object'])

Object Map to Graph（文档对象映射 -> 图）

生成对象映射之后，我们将使用基于可见性的几何算法将对象彼此连接，从而形成一个以对象为节点，连接为边的Graph

几何算法：Connecting objects based on visibility

Input: 
  Object Map
Output: 
  Graph Dictionary: {source_node: [dest_node_1, dest_node_2, …], ..}
  Text List: Text for each node

步骤1：从左上角的对象开始。 对于map中的每个对象，遍历所有其他对象。

步骤2：对于每个对象，向右看。

步骤3：如果另一个对象直接可见，请建立连接。

步骤4：对于每个对象，直接在下面看。

步骤5：如果在直接可见性中存在另一个对象，请建立连接。

实验

所有这些方法都给出了令人鼓舞的结果。 例如，以下是样本发票上的半监督分类方法的输出：

由于半监督学习的范式表明在学习表示时，图中同时存在标记和未标记的节点，因此在单个发票上以图的形式观察到此输出。

共有四个类别-信息，联系人，表格项和其他（未显示）。 最初为每个类标记了一组种子节点。然后，GCN能够从这些初始种子节点中学习未标记节点的表示形式。

可以通过以下方法实现端到端解决方案：首先使用标准NLP技术标识种子节点，然后将Graph馈送到网络。 此处最值得注意的功能是能够以极低的延迟为以前看不见的文档生成节点表示形式。

结论

图卷积神经网络被证明在数据采用连接结构的新型应用中越来越有用。 此外，在结构化文档的情况下，具有空间意义的数据可以适应图形结构，然后与GCN一起使用。

可以在静态图上即时使用半监督学习来生成节点表示，而无需大型训练集。 超越普通的CNN以获得非欧几里得数据，为应用研究的新领域打开了令人兴奋的机遇。
